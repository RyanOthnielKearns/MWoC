# Config Files

MWoC stores all configuration and state in `~/.mwoc/`. The directory is created automatically on first run with mode `700` (owner read/write/execute only).

```
~/.mwoc/
├── auth.json        # API keys — mode 600
├── resources.yaml   # Declared resources — human-editable
├── state.json       # Last-probed state — auto-generated, do not edit
├── bench/           # Benchmark results from `mwoc bench`
│   └── <id>.json
└── evals/           # Cached eval/metadata from HuggingFace and Arena
    ├── <model-hash>.json
    ├── _arena-text.json
    └── id-map.json
```

---

## `auth.json`

Stores API keys for cloud providers. Created and managed by `mwoc auth` commands. Never committed to git.

**Mode:** `600` (owner read/write only)

**Format:**
```json
{
  "anthropic": {
    "apiKey": "sk-ant-..."
  },
  "openai": {
    "apiKey": "sk-..."
  }
}
```

Keys are the provider name as used in `mwoc auth add <provider>`. Currently only `apiKey` is stored per provider; OAuth token support is planned.

### Environment variable fallback

If a provider key isn't in `auth.json`, MWoC falls back to these environment variables:

| Provider | Environment variable |
|---|---|
| `anthropic` | `ANTHROPIC_API_KEY` |
| `openai` | `OPENAI_API_KEY` |
| `google` | `GOOGLE_API_KEY` |

## `resources.yaml`

The source of truth for which resources MWoC knows about. Created by `mwoc init`, updated by `mwoc resource remove`. You can also edit it directly — it's plain YAML.

**Full example with all resource types and tier overrides:**

```yaml
resources:
  # Local machine running Ollama
  - type: local
    name: local-ollama
    backend: ollama
    endpoint: http://localhost:11434
    hardwareNotes: "M3 Max, 128GB unified memory"

  # Web subscription — tracked for awareness, not probeable
  - type: cloud
    name: claude-pro
    provider: anthropic
    tier: Pro
    webOnly: true

  # Anthropic API access — probed using key from auth.json
  - type: cloud
    name: anthropic-api
    provider: anthropic
    tier: API

  # ChatGPT web subscription — tracked for awareness
  - type: cloud
    name: chatgpt-edu
    provider: openai
    tier: Edu
    webOnly: true

  # OpenAI API access
  - type: cloud
    name: openai-api
    provider: openai
    tier: API

  # Remote server, directly reachable over VPN
  - type: server
    name: gpu-rig-1
    backend: vllm
    endpoint: http://10.0.0.1:8000
    accessMethod: direct
    hardwareNotes: "8× A100"

  # Remote server requiring SSH tunnel
  - type: server
    name: lab-server
    backend: sglang
    endpoint: http://localhost:8001
    accessMethod: ssh-tunnel
    sshHost: gpu-box.university.edu
    sshUser: rk123
    localPort: 8001

# Optional: override the built-in tier assignments
tierOverrides:
  my-custom-model: frontier
  llama3.1:8b-instruct-q8_0: mid
```

See [Resource Types](resource-types.md) for full field documentation per type.

## `state.json`

Auto-generated by `mwoc probe`. Contains the last-probed snapshot of all resources and their models. Do not edit this file — it will be overwritten on the next probe.

**Shape:**
```json
{
  "probedAt": "2026-04-11T13:33:46.513Z",
  "resources": [
    {
      "resource": { "type": "local", "name": "local-ollama", ... },
      "status": "available",
      "models": [
        {
          "modelId": "llama3:8b",
          "tier": "local-large",
          "description": "Ollama model: llama3:8b",
          "contextWindow": 8192
        }
      ],
      "probedAt": "2026-04-11T13:33:46.513Z"
    },
    {
      "resource": { "type": "cloud", "name": "anthropic-api", ... },
      "status": "unavailable",
      "models": [],
      "probedAt": "2026-04-11T13:33:46.513Z",
      "error": "No API key configured. Run: mwoc auth add anthropic"
    }
  ]
}
```

`status` per resource is one of `available`, `unavailable`, or `unknown`. Web-only subscriptions always return `unknown` since they cannot be probed.

## `bench/`

Created automatically on the first `mwoc bench` run. Each run is stored as `{id}.json`, where `id` is derived from the run timestamp, resource name, and model ID (colons and slashes replaced with underscores).

These files are the source of truth for `mwoc bench --list` and `mwoc bench --compare`. They are safe to delete manually — the directory will be recreated on the next run.

---

## `evals/`

Cache directory for model metadata and evaluation scores fetched from external sources. All files have a 24-hour TTL (checked by modification time) except `id-map.json`, which has no expiry and accumulates mappings across sessions.

| File | Contents |
|---|---|
| `<model-hash>.json` | HuggingFace eval results and Ollama `/api/show` metadata for one model |
| `_arena-text.json` | Full Chatbot Arena text leaderboard (one file for all models) |
| `id-map.json` | Cached Ollama model ID → HuggingFace model ID mappings |

Delete any file in this directory to force a fresh fetch on next access.
